Openclassrooms Project 8 ( Todo-List ) - Technical Documentation

Product information

Todo List is a tool that helps you organise your life.
It is easy to use and does not require an internet connection.

Features

  • Add todo
  • Edit todo
  • Delete todo
  • Mark a todo as completed
  • Toggle all todos as completed
  • Filter todos
  • Clear all completed todos

Adding a todo

Click on the input field 'What needs to be done?' and enter the title of you new todo

Editing a todo

Double click on the title of the item you would like to edit

Deleting a todo

Hover over the title of the item you would like to edit, a close icon appear. By clicking on the icon the item is deleted.

Marging a todo as completed

Click on the check symbol in the item field, the check symbol turns green and the todo is marked as completed. is deleted.

Toggling all todos to complete

Click on the arrow in the input field, all items will be toggled as completed. is deleted.

Filtering todos

Click on one of the buttons below the list, the list will update an only show items of the chosen category.

Clear all completed todos

Click on one of the 'Clear completed' button below the list, all completed items will be deleted. category.

Technical Information

Code base

This application was built using HTML, CSS, JavaScript and the TodoMVC framework.

The TodoMVC Framework

MVC stands for Mode-View-Controller and is an acrchitectural pattern thatdivides an application into 3 seperated entities. The Model component handles all data-related operations. The View component handles the render-related tasks and includes elements the user interacts with. the Controller component is the interface between Model and Controller component, it handles incoming requests, handles data via the Model component and renders the final output using the View component.

Saving data

The users data is stored locally in the browsers local storage allowing the app to be used without active internet connection.

Set Up

Simply download or clone the repository from my github repository and open the index.html file in your preferred browser (chrome is recommended).

Overall

Using the google chrome devtools and auditing the website these are the overall results:

Performance

Opportunities
  • Next-gen formats: Images, especially icons, can be replaced with smaller image formats such as svg. Google recommends replacing the background image with a next gen image such as webP or jpeg2000. When using those formats one needs to make sure that browser compatibility is given (e.g. safari does not support webP)
  • Render-blockong elements: Elements should, if possible, be loaded at the bottom of the body in order not to block the rendering of the page.
Diagnostics
  • Font-display CSS feature: Ensures the display of text before the font is completely loaded and replaces a standard font once the custom font is loaded.
  • Cache elements: Saves elements such as images and files temporarily in order to increase page laod when revisiting the page.

Accessibility

  • Contrast ratio between background and foreground colors: Text is hardly legible.
  • IDs of elements should be unique
  • Add titles to frame and iframe elements, add alt attributes to images to make it easier for visually impaired people to navigate through the page.
  • Add a language attribute to the html-tag

Best Practices

  • HTTPS: The page does not use a secure transfer protocol. This is important to protect users.
  • Avoid document.write()-method to make sure that even user with slow internet connections do not experience page load delays.
  • Vulnerable third-party scripts: Todolistme uses an old version of jQuery. Always make sure to sue up-to-date libraries without any vulnerabilities to protect users.

SEO

  • meta-tags: missing viewport meta-tag to ensure optimal display in mobile devices.
  • Legible font-sizes: Uses text smaller than 12px. (Application relevant texts are properly sized)
  • Target elements too small: Some clickable elements are too small which makes interaction witht the page harder.

More potential for better performance and cleaner code

  • Inline-JavaScript: Todolistme loads several scripts in the body and also has inline scripts that should be combined into one js-file and then minified.
  • Although todolistme is working on dedicated mobile versions (android and ios) the web app is not repsonsive which is very important these days.
