Openclassrooms Project 8 ( Todo-List ) - Technical Documentation

Product information

Todo List is a tool that helps you organise your life.
It is easy to use and does not require an internet connection.

Features

  • Add todo
  • Edit todo
  • Delete todo
  • Mark a todo as completed
  • Toggle all todos as completed
  • Filter todos
  • Clear all completed todos

Adding a todo

Click on the input field 'What needs to be done?' and enter the title of your new todo.

Editing a todo

Double click on the title of the item you would like to edit.

Deleting a todo

Hover over the title of the item you would like to edit, a close icon appears. By clicking on the icon the item is deleted.

Marking a todo as completed

Click on the check symbol in the item field, the check symbol turns green and the todo is marked as completed.

Toggling all todos to complete

Click on the arrow in the input field, all items will be toggled as completed.

Filtering todos

Click on one of the buttons below the list, the list will update and only show items of the chosen category.

Clear all completed todos

Click on one of the 'Clear completed' button below the list, all completed items will be deleted.

Technical Information

Code base

This application was built using HTML, CSS, JavaScript and the TodoMVC framework.

The TodoMVC Framework

MVC stands for Model-View-Controller and is an acrchitectural pattern that divides an application into 3 seperated entities. The Model component handles all data-related operations. The View component handles the render-related tasks and includes elements the user interacts with. The Controller component is the interface between Model and View component, it handles incoming requests, it handles data via the Model component and renders the final output using the View component.

Saving data

The users data is stored locally in the browsers local storage allowing the app to be used without active internet connection.

Our page performance

I have analyzed the performance of our website

Overall

Using the google chrome devtools and auditing the website these are the overall results:

Performance

Opportunities
  • Render-blocking elements: Elements should, if possible, be loaded at the bottom of the body in order not to block the rendering of the page.
Solution
  • Render-blocking elements: I have moved less important css files to the bottom of the html body tag and saved an estimated load time of .72s

Accessibility

Opportunities
  • Contrast ratio between background and foreground colors: Text is hardly legible.
  • Add labels to input element
Solutions
  • Contrast ratio between background and foreground colors: I changed the font-color of the p tags in the footer and changed the opacity of the h1 tag to .65 (before .15).
  • Add labels to input element: The style of the app does not allow a label here, the placeholder attribute of the input is used in place of a label.

Best Practices

Opportunities
  • HTTPS: The page does not use a secure transfer protocol. This is important to protect users.
  • Error log in console: A file could not be found.
Solutions
  • HTTPS: Here the page is served locally.
  • Error log in console: Remove error from console by fixing the code.

SEO

Opportunities
  • meta-tags: missing viewport meta-tag to ensure optimal display in mobile devices.
  • Legible font-sizes: Uses text smaller than 12px.
  • Target elements too small: Some clickable elements are too small which makes interaction with the page harder.
  • Page does not have a meta description. Meta description are displayed in search results and help users find what they are looking for.
Solutions
  • meta-tags: Add viewport meta-tag
  • Legible font-sizes: Change font size in footer-section.
  • Target elements too small: Was fixed by adding viewport meta-tag.
  • Page does not have a meta description. Added meta description.

Competitors performance

I have analyzed the performance our biggest competitor todolistme.net

Overall

Using the google chrome devtools and auditing the website these are the overall results:

Performance

Opportunities
  • Next-gen formats: Images, especially icons, can be replaced with smaller image formats such as svg. Google recommends replacing the background image with a next gen image such as webP or jpeg2000. When using those formats one needs to make sure that browser compatibility is given (e.g. safari does not support webP)
  • Render-blocking elements: Elements should, if possible, be loaded at the bottom of the body in order not to block the rendering of the page.
Diagnostics
  • Font-display CSS feature: Ensures the display of text before the font is completely loaded and replaces a standard font once the custom font is loaded.
  • Cache elements: Saves elements such as images and files temporarily in order to increase page laod when revisiting the page.

Accessibility

  • Contrast ratio between background and foreground colors: Text is hardly legible.
  • IDs of elements should be unique
  • Add titles to frame and iframe elements, add alt attributes to images to make it easier for visually impaired people to navigate through the page.
  • Add a language attribute to the html-tag

Best Practices

  • HTTPS: The page does not use a secure transfer protocol. This is important to protect users.
  • Avoid document.write()-method to make sure that even users with slow internet connections do not experience page load delays.
  • Vulnerable third-party scripts: Todolistme uses an old version of jQuery. Always make sure to use up-to-date libraries without any vulnerabilities to protect users.

SEO

  • meta-tags: missing viewport meta-tag to ensure optimal display in mobile devices.
  • Legible font-sizes: Uses text smaller than 12px. (Application relevant texts are properly sized)
  • Target elements too small: Some clickable elements are too small which makes interaction witht the page harder.

More potential for better performance and cleaner code

  • Inline-JavaScript: Todolistme loads several scripts in the body and also has inline scripts that should be combined into one js-file and then minified.
  • Although todolistme is working on dedicated mobile versions (android and ios) the web app is not repsonsive which is very important these days.
